Sinn und Zweck dieses Moduls

Normalerweise reicht der in Zen Cart 1.5.7 deutsch integrierte Spamschutz zur Absicherung des Kontaktformulars aus. In 1.5.7 wurde der Spanschutz gegenüber 1.5.6 stark verbessert.
Sollten Sie über das Kontaktformular des Shops trotzdem Spam bekommen, dann können Sie es mit diesem Modul einfach mit einem zusätzlichen Google reCaptcha Version 3 absichern.
reCaptcha v3 ist für den Besucher "unsichtbar", es gibt also keine "Ich bin kein Roboter" Challenges zum Ankreuzen.
Dieses Modul sichert die Seiten Kontakt und optional Artikelbewertung schreiben, Kundenkonto erstellen automatisch mit reCaptcha V3 im Hintergrund ab.

Voraussetzung:
kostenlose Registrierung auf
https://www.google.com/recaptcha/admin

Wichtiger Hinweis für Benutzer des reCaptcha v2 Moduls:
Falls Sie bereits das Google reCaptcha v2 Modul (mit "Ich bin kein Roboter")in Ihrem Zen Cart Shop verwenden, dann entfernen Sie es zunächst komplett, bevor Sie dieses v3 Modul installieren.
Die Vorgangsweise zum Entfernen des alten Moduls ist unter "Update" beschrieben.

Installation

Voraussetzungen


Dieses Modul ist nur für Zen Cart 1.5.7 deutsch geeignet

Empfehlung:

Dieses Modul erst in einem Testsystem einbauen und dort konfigurieren/testen und endgültig an die eigenen Wünsche anpassen. Erst dann in einem Liveshop einsetzen! Sichern Sie unbedingt alle Dateien Ihres Shops per FTP und sichern Sie die Datenbank mit phpMyAdmin oder anderen geeigneten Tools!

WICHTIG
Vor dem Einbau dieser Änderungen:
BACKUP von Shop und Datenbank machen!
Keine Haftung, Verwendung auf eigene Gefahr!
BACKUP gemacht? Ok, dann weiterlesen...

Die Installation erfolgt in folgenden Schritten. Halten Sie diesen Ablauf ein!

1)
Falls Sie noch keine reCaptcha Registrierung haben, registrieren Sie die Domain des Shops zunächst bei Google reCaptcha:
https://www.google.com/recaptcha/admin
Wählen Sie dort als reCaptcha Typ die reCaptcha Version 3 aus und kreuzen Sie an:
Anfragen anhand einer Punktzahl verifizieren.
Sie bekommen dann Websiteschlüssel und Geheimen Schlüssel angezeigt

2)
Stellen Sie nun sicher, dass Sie jedwege andere Google reCaptcha v2 Version komplett aus Ihrem Shop entfernt haben!
Gehen Sie dazu so vor wie unter Update beschrieben.

3)
Im Ordner NEUE DATEIEN den Ordner DEINADMIN auf den Namen Ihres Adminverzichnisses umbenennen
Den Ordner DEINTEMPLATE auf den Namen des in Ihrem Shop aktiven Templates umbenennen (z.B. responsive_classic)
Dann alle Dateien/Ordner aus dem Ordner NEUE DATEIEN in der vorgegebenen Struktur in die Zen Cart Installation hochladen.
Dabei werden keine bestehende Dateien überschrieben.

4)
In die Zen Cart Administration einloggen und auf irgendeinen Menüpunkt clicken.
Sie sollten grüne Erfolgsmeldungen sehen und unter Konfiguration ist der neue Menüpunkt Google reCaptcha V3 Konfiguration vorhanden.
Tragen Sie hier Ihren Websiteschlüssel und Geheimen Schlüssel ein und aktivieren Sie das Modul.
Die anderen Einstellungen dort müssen Sie nicht ändern, das reCaptcha wird dann automatisch auf der Seite Kontakt genutzt.

5)
Prüfen Sie nun in Ihrem Shop, ob unten rechts das Google reCaptcha Badge erscheint und testen Sie die Funktionalität des Kontaktformulars

6)
Die durchgehende Anzeige des Google reCaptcha Badge rechts unten ist meist nicht erwünscht.
Google erlaubt das Entfernen dieses Badges, es sollte aber dann z.B. auf der Kontaktseite ein entsprechender Hinweis plaziert werden, z.B.
"Diese Website ist durch reCAPTCHA geschützt und es gelten die Datenschutzbestimmungen und Nutzungsbedingungen von Google."
Stellen Sie sicher, dass Sie in Ihrer Datenschutzerklärung auf die Nutzung von Google reCaptcha hinweisen.
Um die Anzeige des reCaptcha Badge zu deaktivieren, fügen Sie nun exakt folgende Definition in Ihr Stylesheet hinzu, z.B. in includes/templates/responsive_classic/css/stylesheet.css:

.grecaptcha-badge { visibility: hidden; }

Geändertes Stylesheet hochladen und die Seite neu laden oder gleich Browsercache leeren.


7) OPTIONAL
Falls Sie in Ihrem Shop Spamregistrierungen bekommen oder Spambewertungen, dann können Sie zusätzlich die Seiten Kundenkonto erstellen und Bewertung schreiben mit dem reCaptcha absichern. Standardmäßig ist das absolut nicht notwendig, machen Sie das nur, falls Sie wirklich solche Spamregistrierungen und/oder Spambewertungen bekommen.
Um diese Seiten zu aktivieren, öffnen Sie folgende Datei mit einem Textediitor:
includes/classes/google_recaptcha_v3.php
Ab Zeile 36 finden Sie die Notifier, die berücksichtigt werden können-
Kommentieren Sie hier die aus, die Sie zusätzlich nutzen wollen, z.B. create_account und /oder review_text
Tragen Sie nun zusätzlich in der Modulkonfiguration unter Google reCaptcha V3 - Aktion je Formularseite die entsprechenden Formularseiten ein, z.B.
create_account=default, product_reviews_write=default

UPDATE VON ÄLTEREN RECAPTCHA VERSIONEN

Falls Sie bereits das reCaptcha Modul v2 3.6.1 mit dem "Ich bin kein Roboter Kästchen" in Ihrem Zen Cart Shop verwenden und nun auf v3 umsteigen wollen, dann entfernen Sie das alte Modul zunächst komplett, bevor Sie diese v3 Version 1.1.0 installieren:

Löschen Sie folgende Dateien aus Ihrer Zen Cart Installation:

includes/classes/observers/auto.google_recaptcha_observer.php
includes/classes/observers/google (gesamter Ordner)
includes/functions/extra_functions/functions_google_recaptcha.php
includes/languages/german/extra_definitions/google_recaptcha.php
includes/languages/english/extra_definitions/google_recaptcha.php

Falls vorhanden von einer älteren Version des v2 Moduls:
includes/auto_loaders/config.google_recaptcha.php
includes/classes/observers/class.google_recaptcha.php
includes/functions/extra_functions/recaptchalib.php
includes/languages/german/extra_definitions/reCaptcha.php
includes/languages/english/extra_definitions/reCaptcha.php

Machen Sie die Änderung rückgängig, die das alte Modul vorgenommen hat in:
includes/modules/pages/contact_us/header_php.php
Und stellen Sie die Orginal 1.5.7 Version dieser Datei wieder her

Machen Sie die Änderung rückgängig, die das alte Modul vorgenommen hat in:
includes/templates/DEINTEMPLATE/templates/tpl_contact_us_default.php
Entfernen Sie folgenden Eintrag aus dieser Datei:
<?php echo recaptcha_get_html(false); ?>

Sollten Sie das reCaptcha auch für Registrierung und Bewertung schreiben verwendet haben, prüfen Sie nun noch folgende Dateien:
includes/templates/DEINTEMPLATE/templates/tpl_modules_create_account.php
includes/templates/DEINTEMPLATE/templates/tpl_product_reviews_write_default.php

Entfernen Sie folgenden Eintrag aus diesen Dateien:
<?php echo recaptcha_get_html(false); ?>

Prüfen Sie nun ob Kontaktseite, Bewertungsseite und Registrierungsseite nun wieder problemlos ohne reCaptcha funktionieren

Und dann gehen Sie so vor wie unter Neuinstallation beschrieben.

DEINSTALLATION

Alle von diesem Modul neu angelegten Dateien vom Server löschen.
Danach die uninstall.sql im Ordner UNINSTALL mit einem Texteditor öffnen und den Inhalt ins Eingabefeld unter Tools > SQL Patches Installieren kopieren und absenden.
